# QUICKSTART — from `degit` to first WhatsApp reply

This walkthrough takes you from "I just cloned the template" to "a WhatsApp
message lands at my number and the agent replies". Should take ~15 minutes
once you have the external accounts ready.

## 0. Accounts you'll need

| Service | What for | Where |
|---|---|---|
| Meta Business | A test phone number + app secret | https://developers.facebook.com |
| OpenRouter | One key, routed across many models | https://openrouter.ai/keys |
| A VPS (later) | Production deploy. Skip for the local walkthrough. | Hetzner / DigitalOcean / Linode |

For the local walkthrough you don't strictly need OpenRouter -- the SDK ships
an `EchoLLM` fallback that echoes the buyer's message back. The agent will
respond; the response just won't be model-generated.

## 1. Clone + initialize

```bash
npx degit fmonfasani/whatsapp-sales-saas-template my-saas
cd my-saas
python scripts/init.py
```

`scripts/init.py` prompts for project name / slug / description / author / email /
license / homepage, substitutes the placeholders, renames `src/sample/` to
`src/<your-slug>/`, rewrites lingering `sample.` imports, resets git history,
and self-destructs.

## 2. Install + run the gate

```bash
make dev    # creates .venv, pip install -e ".[dev]"
make check  # ruff + mypy --strict + pytest -- everything green from day 1
```

If `make check` is red on a fresh clone, **please file an issue** -- that
shouldn't happen and the template's CI runs the same gate on every push.

## 3. Configure secrets

```bash
cp .env.example .env
```

Then edit `.env`:

| Variable | Where it comes from |
|---|---|
| `OPENROUTER_API_KEY` | https://openrouter.ai/keys |
| `OPENROUTER_MODEL` | e.g. `anthropic/claude-3-haiku` |
| `META_APP_SECRET` | Meta Developer -> App -> Settings -> Basic |
| `META_VERIFY_TOKEN` | Any random string -- you set it on both sides |
| `APP_ENCRYPTION_KEY` | `python -c "from sample.security import generate_key; print(generate_key())"` |
| `APP_DASHBOARD_ORIGINS` | `http://localhost:3000` for local dev |

`KAPSO_GATEWAY_URL` can stay empty for local -- the in-memory gateway
returns deterministic fake send receipts so the agent loop still runs.

## 4. Bring up dependencies + the API

```bash
make up                                                # postgres + redis via docker
uvicorn services.api.main:app --reload --port 8000
```

In another shell:

```bash
curl http://localhost:8000/health
# -> {"status":"ok"}
```

## 5. Register a tenant

Either via the dashboard (next step) or via curl:

```bash
curl -X POST http://localhost:8000/tenants/connect-whatsapp \
  -H "Content-Type: application/json" \
  -d '{"phone_number_id":"PN-LOCAL-1","business_name":"My Test Shop"}'
# -> {"tenant_id":"...","slug":"my-test-shop","is_new":true}
```

That endpoint is idempotent (POSTing the same `phone_number_id` again returns
`is_new: false` with the same `tenant_id`) -- Meta's webhook retries are
absorbed cleanly.

## 6. Run the admin dashboard

```bash
cd dashboard/admin
npm install
npm run dev
# -> http://localhost:3000
```

You'll see tenants list / new / detail / onboard pages. The dashboard talks
to `http://localhost:8000` by default (configurable with `NEXT_PUBLIC_API_URL`).

## 7. First webhook (locally, with curl)

```bash
# Compute the HMAC for the body
body='{"entry":[{"changes":[{"value":{"metadata":{"phone_number_id":"PN-LOCAL-1"},"messages":[{"type":"text","from":"5491100000000","id":"m1","text":{"body":"hola"}}]}}]}]}'
sig=$(echo -n "$body" | openssl dgst -sha256 -hmac "$META_APP_SECRET" | awk '{print $2}')

curl -X POST http://localhost:8000/webhook \
  -H "Content-Type: application/json" \
  -H "X-Hub-Signature-256: sha256=$sig" \
  -d "$body"
# -> "received 1 for my-test-shop"
```

If you set `OPENROUTER_API_KEY`, the agent reply was generated by the model.
If you didn't, the EchoLLM template echoed the buyer's text. Either way the
reply is also persisted in the buyer's memory -- next turn the model sees it.

## 8. Point Meta at your server

For real WhatsApp traffic you need a public URL. Two options:

- **Local tunnel (dev)**: `ngrok http 8000` -- copy the HTTPS URL into Meta's
  webhook callback config + paste `META_VERIFY_TOKEN`.
- **Production VPS**: follow [`docs/DEPLOY.md`](DEPLOY.md). After
  `bootstrap.sh` + `deploy.sh`, your domain serves HTTPS via nginx with
  Let's Encrypt certs and the same endpoints.

## What's next

- Wire your real catalog into Hindsight (`sample.preprocessor`) so the
  `catalog-lookup` skill actually has facts to find.
- Customize the SOUL prompt in `src/<slug>/agent/soul.py` to your brand.
- Add skills under `src/<slug>/skills/` -- they auto-register via
  `SkillRegistry` and become callable from the agent loop.
- Hook the dashboard up to real auth before exposing it on the internet
  (the template ships it CORS-restricted but unauthenticated).
- Read [`docs/DEPLOY.md`](DEPLOY.md) before pointing real traffic at the
  service. Don't run prod without TLS + secrets in env + rate limiting wired.
